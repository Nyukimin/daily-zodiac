## 実施した見直し・補足

### 1. コア原則・プロジェクト構造・技術スタック
- 前回の依頼で既に本プロジェクト（Daily Zodiac）用に書き換え済みのため、今回の変更対象外。

### 2. セクション0（グローバル宣言）
- **補足**: 本プロジェクトは小規模のため、単一タスクでは task agent が実装を兼任する運用も可、と注釈を追加。委託先が不明な場合の確認を促す一文を追加。

### 3. セクション3（ドキュメントルール）
- **見直し**: 例を別プロジェクト（バトル・ユニット）から、本プロジェクト用（仕様書・実装仕様・星座フォーマット）に変更。
- **見直し**: 3.2 を実態に合わせて整理。現在は `docs/` 直下に `仕様書.md`, `実装仕様.md` 等があるため、「直下に配置可」とし、一次仕様・実装仕様のファイル名を明記。サブフォルダ（仕様/実装仕様/調査）は「使う場合」の任意として記載。

### 4. セクション4（技術スタック詳細）
- **補足**: フォールバック時の具体的な扱い（星座単位で例外時、固定文言で出力し全体を止めない）を一文追加。

### 5. セクション2（プロジェクト構造）
- **補足**: ローカル確認方法（`python generate.py` で `site/` 生成、MVP検証の観点）を追記。

### 6. セクション5（Git運用ルール）
- **見直し**: コミットメッセージの例を、占いプロジェクト関連（仕様書・generate.py/pages.yml・JST修正）に変更。

### 7. セクション6（参照ルール）
- **補足**: 一次仕様として `docs/仕様書.md`, `docs/実装仕様.md` を最優先で参照することを明記。
- **補足**: `rules_domain.md` は他プロジェクト由来の記述が含まれる場合があり、その場合は docs/ の仕様を優先する旨を注釈。
