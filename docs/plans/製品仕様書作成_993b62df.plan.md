# 製品仕様書作成計画

## 概要

ユーザーのリクエストと聞き取り調査の結果を基に、Gocha Rumble 3Dの詳細な製品仕様書を作成します。

## 収集した情報の整理

### 確定事項

1. **バトル形式**: サバイバル制（連続出現、常に敵が出現し続ける）
2. **ユニットシステム**: 固定タイプ（近接、遠距離、支援など）+ カスタマイズ

- バトル中リアルタイム進化（敵を倒すと進化ポイントが貯まり、一定数で進化）
- ガチャ要素あり（バトル中に装備・アイテムを獲得、一時的強化）

3. **ゲームオーバー条件**: 全ユニットが倒される
4. **セットアップフェーズ**: チーム編成、初期装備、陣形、難易度選択
5. **リザルト画面**: 生存時間、撃破数、到達ウェーブ、報酬、ランキング、リプレイ
6. **敵の出現**: 段階的に強くなる + 定期的にボスが出現
7. **バトル中の操作**: カメラ操作のみ（完全放置）

### 未決定事項

- プレイヤーの成長・進行要素（現時点では決めきれない）

## 作成する仕様書の構成

### 1. ゲーム概要

- プロジェクト情報
- コアコンセプト
- ターゲットユーザー

### 2. ゲームシステム詳細

- バトルシステム（サバイバル制の詳細）
- ユニットシステム（固定タイプ、リアルタイム進化、カスタマイズ）
- ガチャシステム（バトル中の装備・アイテム獲得）
- 敵システム（出現パターン、難易度上昇、ボス）

### 3. ゲームフロー

- セットアップフェーズ
- バトルフェーズ
- リザルトフェーズ

### 4. UI/UX設計

- セットアップ画面
- バトル画面（HUD、カメラ操作）
- リザルト画面

### 5. 技術要件

- パフォーマンス目標（1,000+ユニット、60FPS）
- 技術スタック
- 非同期PvPシステム

### 6. 未決定事項・今後の検討項目

- プレイヤーの成長・進行要素

## 実装手順

1. 現在の日時を取得してファイル名を決定（`YYYYMMDD_HHMMSS_product_specification.md`）
2. 仕様書を作成（上記の構成に従って詳細に記述）
3. `docs/仕様/`フォルダに保存
